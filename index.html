关于本项目

核心架构

本项目采用了先进的 Actor 模型 架构与下一代图形技术， 确保在任何负载下都能保持丝滑的体验。

  • PixiJS v8 (WebGPU)：优先使用 WebGPU 进行批处理渲染，在保证方波垂直边缘抗锯齿的同时，维持极低的资源开销。
  • Web Workers：物理引擎与渲染循环完全隔离， 主线程的复杂 DOM 操作（如侧边栏动画）不会阻塞示波器渲染。
  • Ring Buffer：使用位运算优化的环形缓冲区，实现 O(1) 复杂度的实时数据写入。

物理与数学原理

  • Marsaglia Polar Method：在独立线程中生成高斯分布随机数， 模拟真实电路中的热噪声（Johnson-Nyquist noise）。
  • 帧率无关 RC 低通滤波：基于 Delta Time 的指数衰减模型。无论帧率波动或卡顿，电压充放电速度始终符合物理时间，无波形变形。
  • 施密特触发器：通过滞回区间（0.6V - 1.0V）实现抗抖动。
  • 亚稳态模拟：模拟建立/保持时间违例时的输出不确定性。

技术栈

  • TypeScript — 严格类型检查，提升代码可靠性
  • Vite — 现代化构建工具，极速热更新
  • PixiJS v8 — 下一代 2D 渲染引擎 (WebGPU/WebGL)
  • Web Workers — 高性能多线程编程
  • Sass (SCSS) — 模块化样式，变量与 Mixin 复用
  • Catppuccin Macchiato — 优雅的配色主题

开发历程

这个项目见证了我从传统前端开发到现代化工程实践的转变。 最初只是一个简单的 HTML + CSS + JavaScript 演示页面，后来逐步迁移到 Vite + TypeScript + SCSS 技术栈。

后续重构引入了 Web Worker 隔离物理计算，并最终迁移至 PixiJS v8，利用 WebGPU 的批处理能力实现了极致的性能表现，这也是我前端工程化能力的一次完整实践。

在这个过程中，我学会了：模块化架构设计、Actor 模型、 ESLint/Prettier 代码规范，以及如何用 Sass 组织复杂样式。
